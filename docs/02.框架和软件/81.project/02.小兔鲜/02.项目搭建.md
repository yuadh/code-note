## `vue3`下 `Vuex`使用

```js
import {useStore} from 'vuex'
export default{
    setup(){
        const store = useStore() //获取到vuex实例
        console.log(store.state.username)
        console.log(store.getters.newName)
        console.log(store.commit('updata'))
        const mutationsFn = ()=>{ //action
            store.dispatch('updataName')
        }
        return {mutationsFn}
    }
}
```

### 模块化使用

**A模块: 不带命名空间**

```js
state:{
    username:'moduleA'
},
getters:{
    newName(state){
        return state.username+'!!'
    }
},
mutations:{
    updataName(state){
        state.username = 'moduleAAAA'
    }
}
```

**B模块：待命名空间**

```js
namespaced:true,
state:{
    username:'moduleB'
},
getters:{
    newName(state){
        return state.username+'!!'
    }
},
mutations:{
    updataName(state){
        state.username = 'moduleAAAA'
    }
}
```

**在vue中的使用**

```vue
<template>
	<!-- 未带命名空间的模块a使用 -->
	<!-- 未带命名空间的模块只有state有独立的存储其余的方法在公共空间 -->
	<p>{{$store.state.moduleA.username}}</p>
	<p>{{$store.getters.newName}}</p>

	<!-- 带命名空间的模块b使用 -->
	<p>{{$store.state.moduleB.username}}</p>
	<p>{{$store.getters['moduleB/newName']}}</p>
</template>
<script>
 export default{
     name:app,
     setup(){
         const store = useStore()
         const mutationsFn = ()=>{
             store.commit('moduleB/updateName')
         }
     }
 }
</script>
```



一般情况下都会使用命名空间的模块

### 第三方插件

第三方插件

`vuex-persistedstate` 可以帮助项目中 `vuex` 作本地持久化处理

默认存储在 `localstore` 

在 `index.js` 文件中引入插件

```js
export default createStore({
    modules:{
        //...
    },
    plugins:[
        createPersistedState({
            //本地存储的名字
            key:'name-project',
            //需要本地持久化的模块
            paths:['user','cart']
        })
    ]
})
```
