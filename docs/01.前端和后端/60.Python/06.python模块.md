使用模块的好处

- 复用性
- 可维护性，避免函数名和变量名的冲突

为了避免模块名重复，python引入了目录来阻止模块重名的方法



每个包目录下都会有一个 `init.py` 的文件，如果不加此文件会被视为普通话的目录

`init.py` 本身是一个模块，它的模块名为 `目录名`

## 使用模块

`import [module_name]`

```python
import sys

def test():
    age = 3
    args = sys.argv
    if age >= 18:
        print('your age is', age)
        print('adult')
    else:
        print('your age is' ,args[1])
        print('teenager')

if __name__=='__main__':
    test()
```

`sys` 模块存储的是命令行的参数

如果该模块自身执行，`__name__` 会等于 `__main__` ,而引入模块不会执行

## 模块作用域

在模块中，有些函数和变量不希望暴漏，仅仅希望在模块内部使用，可以通过 `_` 前缀实现

```python
def _private_1(name):
    return 'Hello, %s' % name
def _private_2(name):
    return 'Hi, %s' % name
def greeting(name):
    if len(name) > 3:
        return _private_1(name)
    else:
        return _private_2(name)
```

代码段中， 前两个函数是未公开的

但是如果想要使用其中的函数，可以使用类似闭包的。。公开的函数区使用为公开的函数

代码的封装和抽象，无需关心内部的细节
